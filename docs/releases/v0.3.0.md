# CCR v0.3.0

## Highlights

- Capability facts are now presented as an explicit truth stack: local
  overrides, provider discovery, then recognized provider-model hints.
- Registered aliases remain visible in Claude Code's `/model` picker when safe
  for the launch, with no silent fallback for unknown or unsupported routes.
- Responses-only aliases are offered only when their OpenAI-compatible provider
  is explicitly configured as Responses-capable, so the picker cannot select a
  route that the gateway would reject.
- Responses provider statuses that are failed, cancelled, or incomplete for a
  reason other than `max_output_tokens` are surfaced as gateway errors instead
  of synthesized successful assistant turns.
- Provider control rows and non-routable Responses-only aliases are excluded
  from routing; aggregate Doctor and conformance report them as skipped without
  contacting them.
- Computer-use automation is documented as either client-managed by Claude Code
  or opt-in managed execution through Docker, trusted host browser, external
  executor, or unsigned macOS helper preview.
- Release publication is draft-first. Homebrew updates require a separate
  promotion step after archive, checksum, provenance, and GHCR browser-image
  verification.

## User-Visible Changes

- `ccr model show <alias> --json` is the supported way to inspect effective
  capabilities, their sources, discovered values, and local overrides.
- Vision requests require effective image-input support. Unsupported image,
  PDF, audio, structured-output, tool, thinking, or computer-use requests fail
  visibly before provider submission.
- CCR records local route, approval, lifecycle, and audit metadata only. It does
  not store prompts, responses, tool arguments, screenshots, page contents,
  transcript paths, authorization headers, raw hook bodies, or provider secret
  values.
- Redacted route and lifecycle metadata is bounded to 30 days and 10,000
  combined events unless purged earlier with `ccr trace purge --all --yes`.
- The macOS computer-use helper is source-built only. It is not included in
  Homebrew or GoReleaser archives, signed, notarized, or described as
  production-hardened. The preview helper requires macOS 14 or later because
  screenshot capture uses ScreenCaptureKit.

## Compatibility and Limits

- A normal `ccr launch` keeps Claude Code's startup model and subscription auth,
  then adds safe registered aliases to the picker. `chat-only` or tool-disabled
  aliases should be launched directly.
- Subagents, workflow agents, and teammates created after a model switch use the
  active model where Claude Code permits it. Existing workers can remain on
  their spawn-time model and are reported that way.
- Direct first-party Anthropic CUA belongs to Claude Code. OpenAI Responses
  computer use is explicit and managed by CCR: it requires a launch-scoped
  executor, an OpenAI Responses-capable provider, and a route with effective
  Responses and computer-use support. Without the executor, CCR rejects the
  request before provider submission.
- If an OpenAI Responses provider returns `pending_safety_checks`, CCR rejects
  the computer call before execution. It never sends
  `acknowledged_safety_checks` without an approval surface that can show those
  checks to the user.
- The Docker browser image is published to GHCR and signed during release. Host
  browser and external executors are local trust decisions. External executors
  require public HTTPS, no URL credentials/query/fragment/redirects, and
  `--ccr-cua-external-token-env`. The macOS helper is source-built only and
  unsigned.

## Verification

Release evidence is recorded in `docs/acceptance/v0.3.0.md`. Required coverage
includes deterministic Go gates, the 12-job fixture live E2E matrix across
Linux/macOS, pinned/latest Claude Code, and `openai-chat`,
`anthropic-native`, and `openai-responses` fixtures, the default local
real-provider routing matrix, opt-in `test-live-real-full` vision/CUA/executor
evidence where configured, draft-release artifact checks, GitHub provenance
verification, signed GHCR browser-image verification, and post-promotion
Homebrew validation. Skipped real tests are not evidence.
