# CCR v0.2.1 Acceptance Criteria

This is the release source of truth for registered-model picker visibility,
model capability discovery, all-model diagnostics, and the required live
matrix. A skipped live test or an unsupported command is not evidence.

## Picker and Routing

- [x] A normal `ccr launch` starts Claude Code once through the fixed local
  gateway and exposes every safe, registered alias in the same session's
  `/model` picker.
- [x] Claude family substrings are selectively escaped without changing stored
  aliases, first-party model rows, or legacy gateway routing.
- [x] An effective context window of at least 1,000,000 tokens adds exactly one
  terminal `[1m]` marker to discovery, launch allowlists, and printed picker
  commands; smaller or unknown windows do not.
- [x] Canonical escaped picker IDs and legacy gateway IDs, with or without a
  valid suffix, resolve to the same alias. Malformed suffixes fail visibly and
  never fall through.
- [x] New subagents, workflow agents, and teammates use the selected route where
  Claude Code permits it; traces record the route actually used.

## Model Capabilities

- [x] SQLite schema v5 migrates transactionally and preserves all prior rows
  while adding normalized discovery, override, and refresh fields.
- [x] Team profile schema v2 round-trips normalized capabilities and remains
  backward-compatible with v1 imports without exporting secrets.
- [x] OpenAI-compatible discovery keeps allowlisted model metadata; LiteLLM may
  augment it from `/model/info` without persisting `litellm_params`, raw bodies,
  credentials, or unknown provider fields.
- [x] Virtual control rows and explicitly non-chat model kinds are filtered from
  import and excluded from discovery, launch, testing, conformance, and routing.
  Manual capability overrides may mark an existing alias non-routable so an
  inaccurate or unavailable provider classification remains visible.
- [x] Effective precedence is explicit override, then discovered value, then a
  recognized provider-model hint. Unknown remains distinct from false.
- [x] `ccr model refresh <alias>|--all`, `ccr model show <alias> [--json]`, and
  all documented `ccr model update` capability flags validate strictly and
  preserve prior facts on failed or partial refresh.
- [x] The gateway enforces known model restrictions without silently falling
  back to Anthropic or another provider.

## Conformance and Doctor

- [x] `ccr conformance run --all` is a supported command, runs all non-blocked
  aliases with at most four provider workers, continues after failures, orders
  results deterministically, persists each run, and exits nonzero on any
  required failure.
- [x] `ccr doctor --live --all` identifies the failed check and failure kind,
  reports safe gateway/provider HTTP status codes, bounds evidence, suppresses
  provider bodies and secrets, and prints a directly runnable action.
- [x] Legacy registered provider-control rows receive a removal action. Missing
  models, credentials, and provider HTTP failures receive provider-specific
  discovery or test actions.

## Required Matrix

- [x] `make test-live-fixture` proves OpenAI-compatible and
  Anthropic-compatible routing, `[1m]` picker selection, model switching,
  `conformance run --all`, agents, workflows, hooks, sanitized failures, and
  traces against the installed Claude Code CLI.
- [ ] Pull-request fixture CI passes all eight Linux/macOS x Claude Code
  2.1.209/latest x OpenAI-compatible/Anthropic-compatible jobs without skips.
- [x] `CCR_LIVE_REAL_MATRIX=1 make test-live-real` passes locally against
  first-party Anthropic and every configured non-blocked alias, using each
  alias's capability-aware picker ID.
