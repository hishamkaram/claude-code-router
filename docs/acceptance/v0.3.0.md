# CCR v0.3.0 Acceptance Criteria

This is the release source of truth for capability truth, registered-model
picker behavior, vision limits, computer-use boundaries, no-silent-fallback
behavior, local approval/audit privacy, metadata retention, observability, and
draft-first release promotion. A checked item requires reproducible evidence.
Skipped live tests are not evidence.

## Capability Truth and Overrides

- [x] Effective model facts use the documented precedence: explicit local
  override, provider discovery, then recognized provider-model hint.
- [x] Unknown capability metadata remains unknown and is not treated as support
  or non-support.
- [x] `ccr model show <alias> --json` exposes effective values and sources for
  discovered values and overrides.
- [x] Team profile schema v3 preserves provider Responses capability metadata
  during export and import.
- [x] Capability overrides validate strictly, can be cleared with `auto` or
  clear commands where documented, and do not mutate unrelated facts.

## Picker, Routing, and No Fallback

- [x] Default `ccr launch` preserves Claude Code startup/auth behavior and adds
  safe registered aliases to the launch-scoped `/model` picker.
- [x] `chat-only` and tool-disabled aliases are omitted from unsafe picker use
  and remain usable through explicit startup launches where supported.
- [x] Unknown, malformed, blocked, or capability-unsafe CCR model IDs fail
  visibly and never fall through to Anthropic or another provider.
- [x] Provider control models are excluded from routing and `/model`; aggregate
  Doctor and conformance report them as skipped without provider requests.
- [x] The latest authenticated request remains the route source of truth for
  status, trace, status line, sessions, and agents.

## Vision and Computer Use

- [x] Image input is accepted only when the selected alias effectively supports
  vision/image input and the request can be translated safely.
- [x] Unsupported vision, PDF, audio, structured-output, tool, thinking, and
  computer-use requests are rejected before provider submission.
- [x] Direct first-party Anthropic CUA remains owned by Claude Code; CCR records
  only redacted route/audit metadata.
- [x] OpenAI Responses computer use requires a launch-scoped managed executor
  and is rejected before provider submission when none is configured.
- [x] Managed CUA is explicit and launch-scoped, requires effective
  Responses plus computer-use support on an OpenAI Responses-capable provider,
  and distinguishes Docker browser, trusted host browser, external executor,
  and unsigned macOS helper preview.
- [x] External managed CUA requires public HTTPS, no URL
  credentials/query/fragment/redirects, and
  `--ccr-cua-external-token-env`.
- [x] Documentation and release notes describe the macOS helper only as a
  source-built unsigned preview that is not shipped in Homebrew or release
  archives.

## Privacy, Retention, and Observability

- [x] SQLite, logs, CLI output, JSON output, release artifacts, and profile
  exports contain no raw provider secret, authorization header, prompt,
  response, tool argument, screenshot, page content, transcript path, or raw
  hook body.
- [x] Approval, audit, route, and lifecycle metadata are local, redacted, and
  bounded by 30 days and 10,000 combined events.
- [x] `ccr trace purge --all --yes` removes retained runtime history safely.
- [x] `ccr status`, `ccr trace`, `ccr sessions`, and `ccr agents` expose visible
  degradation when lifecycle hooks or trace persistence are unavailable.

## Supported Command Surface

- [x] Help and docs agree for provider, model, conformance, launch, status,
  trace, sessions, agents, doctor, profile, init, and version commands.
- [x] Missing required inputs, unknown commands, and invalid args fail before
  side effects and provide actionable messages.
- [x] Aggregate refresh, conformance, and Doctor commands emit bounded progress
  while provider work is running and return a complete per-alias result.
- [x] Provider/model commands explain what is stored and where, including secret
  references and SQLite state.
